# SBOMとは？

### 辞書的定義
SBOM（Software Bill of Materials、ソフトウェア部品構成表）とは、ソフトウェア製品を構成するすべてのオープンソースパッケージ、ライブラリ、モジュール、およびそれらの間の依存関係を詳細に記録した明細書です。製造業における部品表（BOM）の概念をソフトウェア開発分野に適用したもので、現代の複雑なソフトウェアサプライチェーン内において構成要素の透明性を確保し、セキュリティ脆弱性を体系的に管理するための標準化された一覧を意味します。

### 実務での活用事例
特定のオープンソースライブラリ（例：Log4j）で深刻なセキュリティ脆弱性が発見された際、セキュリティ運用チームは構築済みのSBOMデータベースを検索し、社内のどのシステムやアプリケーションに該当ライブラリが含まれているかを即座に特定します。これにより、全数調査に要する時間を大幅に短縮し、パッチ適用などの対応優先順位を迅速に決定することができます。

### 関連用語
* **ソフトウェアサプライチェーンセキュリティ (SSCS):** ソフトウェアの設計から配布、運用に至る全プロセスにおけるセキュリティ脅威を管理・制御する体系です。
* **SCA (Software Composition Analysis):** アプリケーションに含まれるオープンソースのセキュリティ脆弱性やライセンスリスクを分析する技術です。
* **SLSA (Supply-chain Levels for Software Artifacts):** ソフトウェア成果物の完全性を保証するためにGoogleが提案したセキュリティフレームワークです。
