昨今のサイバー攻撃は、IDとパスワードという第1の関門を越え、ログインセッションそのものを狙う方向へと進化しています。かつてのハッキングがアカウント情報を特定するレベルに留まっていたとすれば、現在はすでに認証が完了した「トークン」を直接奪取し、多要素認証(MFA)体系まで無力化する段階に達しています。その中心にあるのが、生成型AIを活用して攻撃の精巧さと速度を高めた「AIトークンハイジャック(AI Token Hijacking)」です。

## AIトークンハイジャック、企業セキュリティの新たな脅威

Microsoft Defender Security Researchの最新レポートによると、「EvilTokens」と呼ばれるフィッシング・アズ・ア・サービス(PhaaS)ツールキットを活用した大規模なキャンペーンが捕捉されました。この攻撃の核心は、自動化されたインフラを通じてAIトークンハイジャックを試みる点にあります。攻撃者は生成型AIを用いて、被害者の職務やワークフローに最適化されたパーソナライズされた「餌」を作成し、クリックを誘導します。

このように奪取されたトークンは、企業のCloudインフラ全体へと脅威が拡散する足掛かりとなります。トークンが有効な期間中、攻撃者は別途認証を行うことなく、メールの無断持ち出し、社内ディレクトリの分析、権限昇格などを試みることができます。これは単なるデータ流出を超え、企業内の根本的な「信頼の鎖（Chain of Trust）」が崩壊する深刻な状況を招きます。

![AIトークンハイジャック(AI Token Hijacking) - 大規模なフィッシングキャンペーンを表す、サーバーノードで赤い警告アイコンが点滅するグローバルネットワークの概念図。フラットデザインのエディトリアルスタイル、プロフェッショナルなカラーパレット。](../../../../../source/posts/AI_토큰_탈취(AI_Token_Hijacking)/4cc4071d-1.webp)

## 正常な認証手順を悪用するデバイスコードフィッシング

今回発見された攻撃手法の中で特に注意すべきは、「デバイスコード認証フロー(Device Code Flow)」の悪用です。本来、スマートTVやプリンターのように入力インターフェースが制限された機器での利便性のために設計された機能ですが、攻撃者はこの正常なプロセスに割り込み、ユーザーがフィッシングサイトで提供されたコードを入力するように仕向けます。

ここには「動的コード生成」と「バックエンド自動化」という精巧な技術が動員されています。一般的に15分の有効期限を持つデバイスコードを回避するため、攻撃者はRailway.comのようなPaaSプラットフォーム上で数千個の短期的ポーリングノードを生成します。ユーザーがリンクをクリックした瞬間にリアルタイムでコードを生成し、有効期限をすり抜ける仕組みです。ユーザーがコードを入力した途端にセッションが奪取され、攻撃者のセッションはユーザーの環境から分離されているため、従来のセキュリティモニタリングでは検知が非常に困難です。

## データと命令の境界を崩すプロンプトインジェクション

AIエージェントの普及は、新たなセキュリティ戦線を形成しています。最近、主要なAIモデルのエージェントが「間接的プロンプトインジェクション(Indirect Prompt Injection)」攻撃に対して脆弱であることが明らかになりました。これは攻撃者がAIに直接命令を下すのではなく、AIが読み込む外部データ（GitHubのPRタイトル、メール本文など）に悪意のある指示文を潜ませる手法です。

例えば、セキュリティレビューを実行するAIエージェントが悪性コードを含むPRタイトルを読み込んだ瞬間、その中に隠された命令に従い、現在のセッションのAPIキーやトークンを攻撃者のサーバーへ送信してしまう可能性があります。これはAIモデルが「データ」と「命令」を明確に区別できないという構造的な限界に起因します。ユーザーはAIが業務を遂行していると信じていますが、裏側では静かにトークンハイジャックが進んでいるのです。こうした脆弱性は公式なCVEが発行されずに暗黙的にパッチが適用されるケースが多く、旧バージョンのエージェントを使用している環境はリスクにさらされる可能性が高いといえます。

## ネットワークレイヤーでのリアルタイム対応と防御戦略

AIトークンハイジャックに対応するためには、個々のユーザーの注意だけに頼ってはいけません。ネットワークレイヤーから徹底した検証と監視が行われる多層防御戦略が不可欠です。

この観点において、**Haionnet**のセキュリティソリューションは実務的な防御壁を提供します。専用回線ベースの安定したインフラを基盤に、統合セキュリティ管理(MSSP)を通じて異常なトラフィックフローをリアルタイムで追跡します。攻撃者が主に使用するインフラへのデータ流出や、疑わしいドメインからのアクセスをネットワークレベルで遮断することで、第1次防御線を形成します。特に企業のインフラ特性に合わせた精緻なポリシー設定が可能であり、自動化されたAI攻撃からビジネスの継続性を確保する上で中核的な役割を果たします。

> 「AIはセキュリティの強力な味方ですが、同時に攻撃者にとってはかつてないほど鋭い矛となります。今後はログイン後の『セッションの信頼性』をいかに担保するかが、セキュリティの成否を分けるでしょう。」

## 継続的な検証、ゼロトラストへの転換

AIトークンハイジャックはもはや理論上のシナリオではなく、現実の脅威です。自動化されたツールが絶えずセッションを狙っており、業務効率のために導入したAIエージェントが、かえって攻撃の通路になることもあります。

「一度認証されたセッションは安全である」という過信を捨てる時が来ました。すべての接続を絶えず検証し、ネットワークレベルで強力なセキュリティ統制を並行するゼロトラスト原則を実践しなければなりません。Haionnetのような専門パートナーとともにインフラの基礎を再点検すること、それがAI時代を安全に切り抜ける最も確実な方法です。
