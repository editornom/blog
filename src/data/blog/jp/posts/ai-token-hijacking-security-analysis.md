最近のサイバー攻撃は、IDとパスワードという第1の関門を越え、ログインセッションそのものを狙う方向へと進化しています。かつてのハッキングがアカウント情報を聞き出すレベルに留まっていたとすれば、現在はすでに認証が完了した「トークン」を直接奪取し、多要素認証（MFA）体系まで無力化する段階に達しています。その中心にあるのが、生成型 AI を活用して攻撃の精巧さと速度を高めた「AIトークン・ハイジャッキング（AI Token Hijacking）」です。

## AIトークン・ハイジャッキング、企業セキュリティの新たな脅威

Microsoft Defender Security Researchの最新レポートによると、「EvilTokens」と呼ばれるサービスとしてのフィッシング（PhaaS）ツールキットを活用した大規模なキャンペーンが捕捉されました。この攻撃の核心は、自動化されたインフラを通じてAIトークン・ハイジャッキングを試みる点にあります。攻撃者は生成型 AI を用いて、被害者の職務やワークフローに最適化されたオーダーメイドの「餌」を投げ、クリックを誘導します。

このように奪取されたトークンは、企業の Cloud インフラ全体へと脅威が拡散する足がかりとなります。トークンが有効な期間中、攻撃者は別途の認証なしにメールの無断持ち出し、社内ディレクトリの分析、権限昇格などを試みることができます。これは単なるデータ流出を超え、企業内の根本的な「信頼の連鎖（Chain of Trust）」が崩壊する深刻な事態を招きます。

## 正常な認証手順を突くデバイスコード・フィッシング

今回発見された攻撃手法の中でも、特に注意深く見るべきは「デバイスコード認証フロー（Device Code Flow）」の悪用です。本来、スマートTVやプリンターのように入力インターフェースが制限された機器での利便性のために設計された機能ですが、攻撃者はこの正常なプロセスに介入し、ユーザーがフィッシングサイトで提供されたコードを入力するように仕向けます。

ここには「動的コード生成」と「バックエンド自動化」という精巧な技術が動員されます。一般的に15分の有効期限を持つデバイスコードを回避するため、攻撃者はRailway.comのようなPaaSプラットフォーム上で数千もの短期間ポーリング・ノード（Polling Node）を生成します。ユーザーがリンクをクリックする瞬間にリアルタイムでコードを生成し、有効期限をすり抜ける方式です。ユーザーがコードを入力した瞬間にセッションが奪取され、攻撃者のセッションはユーザーの環境から分離されているため、従来のセキュリティ監視では検知が非常に困難です。

## データと命令の境界を崩すプロンプト・インジェクション

AI エージェントの普及は、もう一つのセキュリティ戦線を形成しています。最近、主要な AI モデルのエージェントが「間接的プロンプト・インジェクション（Indirect Prompt Injection）」攻撃に対して脆弱であることが明らかになりました。攻撃者が AI に直接命令を下す代わりに、AI が読み取る外部データ（GitHubのPRタイトル、メール本文など）に悪意のある指示文を潜ませる手法です。

例えば、セキュリティレビューを実行する AI エージェントが悪性コードを含むPRタイトルを読み取った瞬間、その中に隠された命令に従い、現在のセッションの API キーやトークンを攻撃者のサーバーへ送信してしまう可能性があります。これは AI モデルが「データ」と「命令」を明確に区別できない構造的な限界に起因します。ユーザーは AI が業務を遂行していると信じていますが、裏側では静かにトークン奪取が進んでいるわけです。このような脆弱性は、公式な CVE が発行されずに暗黙的にパッチが適用されるケースが多く、古いバージョンのエージェントを使用している環境は、リスクにさらされたままになる可能性が高いです。

## ネットワークレイヤーでのリアルタイム対応と防御戦略

AIトークン・ハイジャッキングに対応するためには、個々のユーザーの注意だけに頼ってはいけません。ネットワークレイヤーから徹底した検証と監視が行われる多層防御戦略が不可欠です。

このような観点から、**Haionnet**のセキュリティソリューションは実務的な防壁を提供します。専用回線ベースの安定したインフラを土台に、統合セキュリティ監視（MSSP）を通じて異常なトラフィックフローをリアルタイムで追跡します。攻撃者が主に使用するインフラへのデータ流出や、疑わしいドメインからのアクセスをネットワーク段階で元から遮断することで、第1次防衛線を形成します。特に企業のインフラ特性に合わせた精巧なポリシー設定が可能であり、自動化された AI 攻撃からビジネスの継続性を確保する上で中枢的な役割を果たします。

> 「AI はセキュリティの強力な味方ですが、同時に攻撃者にとってはかつてないほど鋭い矛となります。これからはログイン後の『セッションの信頼性』をいかに担保するかが、セキュリティの成否を分けるでしょう。」

## 継続的な検証、ゼロトラストへの転換

AIトークン・ハイジャッキングは、もはや理論上のシナリオではなく現実の脅威です。自動化されたツールが絶えずセッションを狙っており、私たちが業務効率化のために導入した AI エージェントが、かえって攻撃の通路になることもあります。

今こそ「一度認証されたセッションは安全だ」という過信を捨てるべき時です。すべての接続を絶えず検証し、ネットワークレベルでの強力なセキュリティ統制を並行するゼロトラスト原則を実践しなければなりません。Haionnetのようなセキュリティパートナーと共にインフラの基礎を再点検すること、それが AI 時代を安全に切り抜ける最も確実な方法です。

  </div>
</details>

<details>
  <summary>間接的プロンプト・インジェクション(Indirect Prompt Injection)とは何ですか？</summary>
  <div class="faq-content">

AI が読み込む外部データ（メール、文書など）に悪意のある指示文を隠しておく手法です。AI モデルがデータと命令を明確に区別できない構造的限界を突き、AI 自らがセッション・トークンや API キーを攻撃者に送信するように仕向けます。

  </div>
</details>

<details>
  <summary>なぜログイン後の「セッションセキュリティ」が重要になったのですか？</summary>
  <div class="faq-content">

最近の攻撃は第1次認証の突破ではなく、認証の結果物である「セッション」を狙うためです。セッションが奪取されると、攻撃者は追加認証なしで企業の Cloud インフラ全体を操作できるため、根本的な信頼体系が崩壊するリスクが非常に高いです。

  </div>
</details>

<details>
  <summary>攻撃者がデバイスコード回避のために使用する技術的な方法は？</summary>
  <div class="faq-content">

Railway.comのようなPaaSプラットフォームで、数千もの「短期間ポーリング・ノード」を生成します。15分という短い有効時間を克服するため、ユーザーがリンクをクリックする瞬間にバックエンドの自動化でリアルタイムにコードを生成し、認証セッションを即座に釣り上げます。

  </div>
</details>

<details>
  <summary>AI エージェント環境でセキュリティ上留意すべき点は何ですか？</summary>
  <div class="faq-content">

AI エージェントの脆弱性は、公式な CVE が発行されずに暗黙的にパッチが適用されることが多いです。そのため、古いバージョンのエージェントを使用しないよう注意し、AI が処理する外部データが命令に豹変する可能性があることを常に警戒しなければなりません。

  </div>
</details>

<details>
  <summary>Haionnetのソリューションはどのような方式で対応しますか？</summary>
  <div class="faq-content">

専用回線ベースのインフラと統合セキュリティ監視（MSSP）を通じて、ネットワークレイヤーで第1次防衛線を構築します。異常なトラフィックフローをリアルタイムで追跡し、不審なドメインへのデータ流出を元から遮断することで、ビジネスの継続性を確保します。

  </div>
</details>

<details>
  <summary>従来のセキュリティ方式と差別化された多層防御戦略とは？</summary>
  <div class="faq-content">

ユーザーの注意だけに頼らず、ネットワーク側で強力なポリシーを設定することです。リアルタイムのトラフィック分析に加え、インフラ特性に合わせた精巧な検証手順を導入し、自動化された AI 攻撃が内部へ拡散するのを階層ごとに遮断します。

  </div>
</details>
