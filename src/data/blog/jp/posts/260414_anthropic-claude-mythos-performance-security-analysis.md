生成 AI の技術競争の軸が、単なるテキスト生成を超えて、ソフトウェアの核心である「セキュリティ」領域へと急速にシフトしています。最近 Anthropic が公開した新しいモデル「Claude Mythos プレビュー」は、こうした変化を象徴的に示しています。従来のモデルとは一線を画すサイバーセキュリティ分析能力を備え、業界に小さくない衝撃を与えました。Anthropic が発表したデータを基に、なぜ Mythos がセキュリティ実務において重要な転換点となるのか、その技術的な実態と示唆するポイントをまとめました。

## 汎用モデルの進化がもたらしたセキュリティ性能の飛躍的向上

Claude Mythos は、設計段階からセキュリティのみをターゲットに特化訓練を受けたモデルではありません。むしろ、全体的なコード理解度、推論能力、そして自律性を引き上げる過程で、セキュリティ脆弱性攻撃（Exploit）能力が自然と向上したケースに近いと言えます。

前世代のフラッグシップモデルである Claude 3.5 Opus (Opus 4.6) と比較すると、その差は歴然としています。Mozilla Firefox の JavaScript エンジン脆弱性テストの結果、Opus 4.6 は数百回の試行の末、わずか 2 回の成功にとどまったのに対し、Mythos は同一条件で 181 回もの成功的な攻撃コードを生成しました。これは単なる性能改善ではなく、モデルの能力そのものが別次元に進化したことを意味しています。

![Claude OpusとClaude Mythosの脆弱性検知率を比較した棒グラフ](../../../../../source/posts/Anthropic_Claude_Mythos_성능_공개/c9860455-1.png)

### 27 年間隠れていた OpenBSD の欠陥を見つけ出す

今回の発表で最も注目された成果は、セキュリティが高いことで有名なオペレーティングシステム OpenBSD でゼロデイ脆弱性を発見した事例です。Mythos はユーザーから「脆弱性を探してほしい」という単純なリクエストを受け取った後、自らコードを分析し、仮想環境で実験を繰り返すことで、27 年間誰にも発見されなかったバグを見つけ出しました。

この脆弱性の核心は、TCP（Transmission Control Protocol）の選択的確認応答である SACK（RFC 2018）の実装方式にありました。OpenBSD は受信されていないデータ範囲を管理するために単一連結リスト（Singly Linked List）構造を使用していますが、Mythos は特定の条件下で「符号付き整数オーバーフロー（Signed Integer Overflow）」が発生する可能性があることを見抜きました。

> 「Mythos は、攻撃者が送信した SACK ブロックの開始点が実際のウィンドウから約 2^31 離れているとき、カーネルが不可能な条件を満たしたと判断し、ヌルポインタ参照（Null Pointer Dereference）を引き起こしてシステムをクラッシュさせられることを証明しました。」

驚くべき点は、この一連の分析プロセスにかかったコストがわずか 50 ドル未満だったという事実です。熟練した専門家が数週間にわたって取り組むべき作業を、AI が一晩のうちに、しかも極めて安価に成し遂げたのです。企業にとってはセキュリティ監査コストを劇的に下げるチャンスですが、逆に悪意のある攻撃者が手にできる武器がそれだけ強力になったことも意味します。

### FFmpeg と仮想化レイヤーまで及ぶ分析能力

Mythos の能力はカーネル領域にとどまりません。世界中の多くのメディアサービスの基盤となっている FFmpeg ライブラリにおいても、16 年前の脆弱性を発掘しました。H.264 コーデックのスライス（Slice）カウンター処理過程で発生したこのバグは、これまで数多くのファジング（Fuzzing）ツールや人間のレビュアーが見逃してきた部分でした。

Cloud 環境の中核である仮想マシンモニタ（VMM）のメモリ破壊脆弱性を見つけ出したプロセスも興味深いものです。Mythos は Rust や Java のようにメモリ安全性を強調する言語で書かれたプロジェクトでも隙間を見つけ出しました。ハードウェア制御のために不可欠な `unsafe` キーワードやローレベルのポインタ操作箇所を正確に攻略したのです。これは、Mythos が単にコードパターンを暗記しているのではなく、プログラムの実行フローとメモリ構造を実質的に理解していることを示しています。

## Project Glasswing：防御効率を最大化するための試み

Anthropic は、Mythos の強力な性能が悪用される可能性を警戒し、「Project Glasswing（プロジェクト・グラスウィング）」を同時に開始しました。モデルを無差別に一般公開する前に、主要な産業パートナーやオープンソースのメンテナーにまず提供し、基幹ソフトウェアのセキュリティ上の欠陥を先制的に補完するという趣旨です。

彼らの戦略は「防御者の優位（Defender's Advantage）」を先占することにあります。短期的には攻撃者に有利に見えるかもしれませんが、長期的にはコードがデプロイされる前に AI がすべてのバグを修正する時代の到来を早めることができるという信念に基づいています。もちろん、この技術が普及する過渡期には、少なからぬ混乱が予想されます。

ビジネスの観点からは、IT インフラ管理のパラダイムシフトが求められます。もはやファイアウォールの設定といった伝統的な手法だけでなく、使用しているすべてのソフトウェアスタックをリアルタイムで検証しパッチを適用する、高度な体系が不可欠です。

### 実務的観点のセキュリティ戦略とインフラの結合

Claude Mythos の登場は、私たちに明確な課題を突きつけています。使用しているライブラリの安全性をいかに継続的に検証するか、そしてこのような高性能 AI をセキュリティ業務のプロセスにいかに内在化させるか、という悩みです。

実務的に最も急務なのは、セキュリティアップデートの自動化と、ネットワークレベルでの徹底した隔離です。特に B2B 環境では、小さなセキュリティ事故でも企業の信頼性に致命的なダメージを与えるため、専門的なネットワークセキュリティパートナーシップを通じた防御体系の構築がますます重要になっています。

安定したネットワークセキュリティソリューションを備えることで、企業は本来のビジネスに集中できる環境を整えられます。Mythos のような高知能 AI の脅威が顕在化しても、企業ネットワークと Cloud 環境が物理的・論理的に強固に保護されていれば、被害を最小限に抑えることができます。防御の効率を高めるために、最新の技術ツールと安定したインフラを組み合わせるハイブリッド戦略が、今こそ必要とされています。

## AI セキュリティ時代、実質的な脅威と対応の始まり

Anthropic Claude Mythos の性能公開は、単なる技術アップデート以上のメッセージを込めています。サイバーセキュリティの攻防メカニズムが AI を中心に完全に再編される特異点（シンギュラリティ）に突入したことを知らせる号砲です。

自律的にゼロデイ脆弱性を分析し、エクスプロイトまで作成する AI の登場は確かに脅威ですが、同時にシステムを完璧に近い状態まで磨き上げられる機会でもあります。結局、この技術を誰がより責任を持って、そして迅速に活用できるかが鍵となるでしょう。

現場のエンジニアや意思決定者は、もはや AI を脅威として退けるのではなく、システムの防御力を最大化する強力なパートナーとして受け入れるべきです。その過程で、信頼できるセキュリティインフラの基盤が支えとなれば、変化するセキュリティパラダイムの中でも安定したビジネス運営が可能になるはずです。今後続く Anthropic の追加データやパッチ適用の動向を注視し、次世代のセキュリティ戦略を具体化すべき時です。
