生成式 AI 的技术竞争轴心正迅速从单纯的文本生成转向软件的核心领域——“安全”。最近 Anthropic 公开的新模型 “Claude Mythos Preview” 正是这一变化的象征。它具备了与以往模型截然不同的网络安全分析能力，在业内引起了不小的轰动。基于 Anthropic 发布的数据，我们整理了 Mythos 为何会成为安全实务环境中的重要转折点，及其技术实质与启示。

## 通用模型的进化带来安全性能的飞跃

Claude Mythos 并非从设计阶段就仅针对安全领域进行特化训练的模型。相反，它更像是在提升整体代码理解力、推理能力和自主性的过程中，自然而然地带动了安全漏洞攻击 (Exploit) 能力的提升。

与前代旗舰模型 Claude 3.5 Opus (Opus 4.6) 相比，其差异显而易见。在 Mozilla Firefox JavaScript 引擎漏洞测试中，Opus 4.6 在数百次尝试后仅成功 2 次，而 Mythos 在相同条件下成功生成了 181 次有效的攻击代码。这不仅意味着性能的改善，更代表了模型本身量级的彻底跨越。

![Claude Opus与Claude Mythos漏洞探测率对比柱状图仪表盘。](../../../../../source/posts/Anthropic_Claude_Mythos_성능_공개/c9860455-1.png)

### 挖掘出隐藏 27 年之久的 OpenBSD 漏洞

此次发布中最引人注目的成果是在以安全性著称的操作系统 OpenBSD 中发现了零日漏洞。Mythos 在收到用户寻找安全漏洞的简单请求后，自主分析代码并在虚拟环境中反复实验，最终找出了 27 年来无人发现的漏洞。

该漏洞的核心在于 TCP (Transmission Control Protocol) 选择性确认 SACK (RFC 2018) 的实现方式。OpenBSD 使用单链表 (Singly Linked List) 结构来管理未接收的数据范围，Mythos 敏锐地观察到在特定条件下会发生 “有符号整数溢出 (Signed Integer Overflow)”。

> “Mythos 证明了当攻击者发送的 SACK 数据块起点距离实际窗口约 2^31 时，内核会判定满足了一个不可能的条件，从而触发空指针引用 (Null Pointer Dereference)，导致系统崩溃。”

令人震惊的是，整个分析过程的成本不到 50 美元。资深专家需要数周才能完成的工作，AI 在一夜之间以极低的成本达成了。对企业而言，这是大幅降低安全审计成本的机会；但反之，这也意味着恶意攻击者拥有的武器变得空前强大。

### 延伸至 FFmpeg 与虚拟化层的分析能力

Mythos 的能力不仅局限于内核领域。它在作为全球众多媒体服务基础的 FFmpeg 库中，也发掘出了一个长达 16 年之久的漏洞。这个发生在 H.264 解码器 Slice 计数器处理过程中的漏洞，此前曾避开了无数模糊测试 (Fuzzing) 工具和人工审计。

在寻找云环境核心组件——虚拟机监控器 (VMM) 的内存损坏漏洞的过程中，Mythos 的表现同样有趣。即便是在 Rust 或 Java 等强调内存安全性的语言编写的项目中，它也能找到破绽。它精准地攻击了为了硬件控制而不得不使用的 `unsafe` 关键字或底层指针操作点。这表明 Mythos 并非单纯死记硬背代码模式，而是实质性地理解了程序的执行流和内存结构。

## Project Glasswing：旨在最大化防御效率的尝试

Anthropic 对 Mythos 强大性能可能被滥用的风险保持警惕，并同步启动了 “Project Glasswing (玻璃翼项目)”。其宗旨是在模型广泛公开前，先提供给主要行业合作伙伴和开源维护者，以便预先修复核心软件的安全缺陷。

他们的战略在于抢占 “防御者的优势”。短期内这似乎对攻击者有利，但从长远来看，这能加速推进“在代码部署前由 AI 先修复所有漏洞”的时代。当然，在这一技术普及的过渡期，预计会出现不小的混乱。

从商业视角看，这要求 IT 基础设施管理范式发生转变。现在，不仅需要防火墙设置等传统方式，更需要能够实时验证和修复我们使用的所有软件栈的高级体系。

![人类分析师与 AI 化身通过全息显示屏协作监控全球网络流量的未来派安全运营中心 (SOC)。](../../../../../source/posts/Anthropic_Claude_Mythos_성능_공개/62d06f8e-4.png)

### 实务视角的安全策略与基础设施结合

Claude Mythos 的出现给我们留下了明确的课题：如何持续验证所使用库的安全性，以及如何将这种高性能 AI 内化到安全业务流程中。

实务上最紧迫的任务是安全更新的自动化，以及网络层面的彻底隔离。特别是在 B2B 环境中，微小的安全事故也会对企业信誉造成致命打击，因此通过专业的网络安全合作伙伴构建防御体系变得愈发重要。

拥有稳定的网络安全解决方案后，企业才能专注于自身业务。即使像 Mythos 这样的高智能 AI 威胁变得具体化，只要企业网和 Cloud 基础设施在物理和逻辑上得到严密保护，就能将损失降至最低。为了提高防御效率，将最新技术工具与稳定基础设施相结合的混合战略，比以往任何时候都更加必要。

## AI 安全时代，实质性威胁与应对的开始

Anthropic Claude Mythos 的性能公开传递了超越技术更新本身的信息。它标志着网络安全攻防机制已进入以 AI 为中心的完全重构的奇点。

能够自主分析零日漏洞并编写 Exploit 的 AI 虽然具有威胁性，但同时也是将系统打磨至近乎完美的机遇。最终，关键在于谁能更负责任、更敏捷地利用这一技术。

一线的工程师和决策者现在不应仅将 AI 视为威胁，而应将其作为最大化系统防御能力的强力伙伴。在此过程中，如果拥有可靠的安全基础设施支撑，即便在变化的攻击范式下，也能实现稳定的业务运营。现在是时候密切关注 Anthropic 的后续数据与补丁动态，具体化下一代安全战略了。
