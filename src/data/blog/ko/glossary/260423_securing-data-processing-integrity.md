faqs:
- q: 기밀 컴퓨팅이란 무엇인가요?
  a: 하드웨어 기반의 신뢰 실행 환경(TEE)을 통해 데이터가 처리되는 동안에도 이를 보호하는 기술입니다. CPU 내에 격리된 공간을 만들어 메모리상의
    데이터를 암호화하고 무결성을 유지합니다.
- q: 기밀 컴퓨팅이 왜 중요한가요?
  a: 기존의 저장 및 전송 중 암호화와 달리, 데이터 연산을 위해 복호화된 상태에서도 보안을 유지하기 때문입니다. 이를 통해 OS나 관리자 권한을
    가진 공격자로부터 데이터를 지킬 수 있습니다.
- q: TEE의 핵심적인 역할은 무엇인가요?
  a: CPU 내부에 일반 프로세스와 완전히 분리된 독립 보안 영역인 엔클레이브를 생성합니다. 인가된 코드만이 하드웨어 레벨에서 데이터를 복호화해
    실행할 수 있도록 물리적으로 격리하는 역할을 합니다.
- q: '''처리 중인 데이터'' 보안은 어떤 의미인가요?'
  a: 메모리에 로드되어 실제 연산이 일어나는 시점의 데이터를 보호한다는 의미입니다. 그동안 보안의 공백이었던 런타임 환경에서도 데이터의 기밀성과
    무결성을 확보할 수 있게 됩니다.
- q: 원격 증명이란 무엇인가요?
  a: 데이터 소유자가 서버로 워크로드를 보내기 전, 해당 하드웨어의 신뢰성과 소프트웨어 변조 여부를 암호학적으로 검증하는 절차입니다. 객관적인
    수치로 시스템의 무결성을 확인할 수 있습니다.
- q: 기존의 암호화 방식과는 어떤 차이가 있나요?
  a: 기존 방식은 데이터가 저장되거나 이동할 때만 보호하지만, 기밀 컴퓨팅은 데이터가 사용되는 순간까지 보호합니다. 특히 인프라 관리자조차 데이터에
    접근할 수 없다는 점이 가장 큰 차별점입니다.
- q: 퍼블릭 클라우드에서도 안전을 보장할 수 있나요?
  a: 그렇습니다. 하드웨어 수준에서 강제적인 격리가 이루어지므로 클라우드 서비스 제공자나 호스트 운영체제조차 엔클레이브 내부 데이터에 접근할 수
    없습니다. 기업은 통제권 밖에서도 보안을 유지합니다.
- q: 데이터 클린 룸 서비스에 어떻게 활용되나요?
  a: 여러 기관이 원본 데이터를 노출하지 않고 공동 분석을 진행할 때 활용됩니다. 기밀 컴퓨팅 환경 내에서 데이터를 결합해 학습하되, 각 참여자는
    상대방의 데이터에 접근하지 못하고 오직 결과물만 공유받습니다.
- q: AI 모델 보호에는 어떤 이점이 있나요?
  a: 수조 원의 가치를 지닌 AI 모델의 아키텍처와 가중치가 추론 과정에서 유출되는 것을 방지합니다. 모델 전체를 기밀 컴퓨팅 환경에 적재하여
    실행함으로써 독점적인 지적 재산을 안전하게 보호할 수 있습니다.
- q: 기밀 컴퓨팅이 지향하는 보안의 최종 형태는 무엇인가요?
  a: 제로 트러스트 원칙을 컴퓨팅 인프라의 심부인 하드웨어 레벨까지 확장하는 것입니다. 데이터 활용의 전 과정에서 완벽한 통제권을 제공하여 보안이
    비즈니스의 가속기로 기능하게 만드는 것을 지향합니다.

[이미지: Hardware-based Trusted Execution Environment (TEE) and encrypted data processing flow]

그동안의 데이터 보안은 크게 두 가지 영역에 집중되어 왔습니다. 디스크나 테이프에 저장된 데이터를 보호하는 '저장 중(At Rest) 암호화'와 네트워크를 통해 오가는 데이터를 지키는 '전송 중(In Transit) 암호화'가 그것입니다. 하지만 실제 연산을 위해 데이터를 복호화하여 메모리에 올리는 순간, 보안의 공백이 발생합니다. 운영체제(OS)의 커널이나 하이퍼바이저, 혹은 시스템 관리 권한을 가진 공격자가 메모리상의 원천 데이터에 접근할 수 있는 위험에 노출되기 때문입니다. 기밀 컴퓨팅(Confidential Computing)은 바로 이 '처리 중(Data in Use)' 상태의 데이터를 보호하기 위한 기술적 대안으로 부상했습니다.

하드웨어 수준의 물리적 격리, TEE의 핵심 원리

기밀 컴퓨팅의 중추는 하드웨어 기반의 신뢰 실행 환경(Trusted Execution Environment, TEE)입니다. 이는 CPU 내부에 일반적인 소프트웨어 프로세스와 완전히 분리된 독립적인 보안 영역, 이른바 '엔클레이브(Enclave)'를 생성하는 기술입니다. 기밀 컴퓨팅 환경에서는 연산에 필요한 데이터가 메모리에 로드될 때 실시간으로 암호화되며, 오직 인가된 특정 코드만이 하드웨어 레벨에서 이를 복호화해 실행할 수 있습니다.

이 과정에서 인프라 관리자나 호스트 운영체제조차 해당 영역 내부의 데이터에 접근하거나 내용을 수정할 수 없습니다. Intel TDX, AMD SEV-SNP, NVIDIA의 기밀 컴퓨팅 아키텍처 등 주요 반도체 제조사들이 제공하는 기술적 명세들은 모두 이러한 하드웨어 수준의 강제적 격리를 지향하고 있습니다.

검증 가능한 신뢰의 기반, 원격 증명

기밀 컴퓨팅이 클라우드 환경에서 실질적인 효용을 갖는 이유는 '원격 증명(Remote Attestation)' 매커니즘 덕분입니다. 이는 데이터 소유자가 민감한 워크로드를 원격지의 서버로 보내기 전, 해당 서버의 하드웨어가 신뢰할 수 있는 상태인지, 그리고 실행될 소프트웨어가 변조되지 않았는지를 암호학적으로 검증하는 절차입니다. 

단순히 인프라 제공자의 선의에 기대는 것이 아니라, 하드웨어가 생성한 디지털 서명을 통해 시스템의 무결성을 객관적으로 증명받는 것입니다. 이를 통해 기업은 자사 통제권 밖에 있는 퍼블릭 클라우드 환경에서도 온프레미스 수준의 데이터 기밀성을 유지하며 보안 정책을 관철할 수 있습니다.

다자간 데이터 협업과 AI 모델 보호로의 확장

기밀 컴퓨팅의 적용 범위는 단순한 보안 강화를 넘어 새로운 비즈니스 모델의 구현으로 확대되고 있습니다. 대표적인 사례가 '데이터 클린 룸(Data Clean Room)'을 활용한 다자간 보안 분석입니다. 금융, 의료 등 데이터 규제가 엄격한 산업군에서 서로 다른 기관들이 원천 데이터를 외부에 노출하지 않은 채 기밀 컴퓨팅 환경 내에서 데이터를 결합하고 공동 학습을 진행할 수 있습니다. 각 참여자는 결과물만 공유받을 뿐, 상대방의 기밀 데이터에는 절대 접근할 수 없습니다.

최근에는 고도화된 AI 모델 아키텍처와 가중치(Weight)를 보호하기 위한 수단으로도 주목받고 있습니다. 수조 원의 비용이 투입된 독점적 AI 모델이 추론 과정에서 유출되는 것을 방지하기 위해, 모델 전체를 기밀 컴퓨팅 환경 내에 적재하여 실행하는 방식이 실무에 적용되는 추세입니다.

결과적으로 기밀 컴퓨팅은 제로 트러스트(Zero Trust) 원칙을 컴퓨팅 인프라의 심부인 하드웨어 레벨까지 확장하는 역할을 수행합니다. 데이터가 전송되고 저장되는 단계를 넘어, 실제 활용되는 런타임 환경에서도 완벽한 통제권을 유지하고자 하는 기술적 지향점은 보안이 비즈니스의 제약이 아닌 가속기로 기능하게 만드는 핵심 동력이 될 것입니다.
