faqs:
- q: 거대 언어 모델(LLM)이란 무엇인가요?
  a: LLM은 방대한 양의 텍스트 데이터를 학습하여 인간의 언어를 이해하고 생성하도록 설계된 인공지능 모델입니다. 수억 개 이상의 파라미터를 바탕으로 단순한 문장 작성을 넘어 요약, 번역, 복잡한 논리 추론까지 수행할 수 있는 것이 특징입니다.
- q: LLM의 핵심이 되는 '트랜스포머' 아키텍처는 어떤 역할을 하나요?
  a: 트랜스포머는 '자기 어텐션(Self-Attention)' 메커니즘을 통해 문장 내 모든 단어 사이의 관계를 동시에 계산하는 기술입니다. 이를 통해 문장이 길어져도 앞부분의 정보를 잊지 않고 문맥을 정확하게 파악하며, 데이터 병렬 처리를 통해 학습 속도를 획기적으로 높여줍니다.
- q: LLM에서 말하는 '창발적 능력(Emergent abilities)'이란 무엇인가요?
  a: 모델의 파라미터 수와 데이터 규모가 특정 임계치를 넘어설 때, 이전의 작은 모델에서는 볼 수 없었던 고차원적인 추론 능력이 갑자기 나타나는 현상을 말합니다. 이는 확장 법칙(Scaling Law)에 따라 컴퓨팅 자원과 데이터가 커질수록 모델의 역량이 비약적으로 향상되는 과정에서 발생합니다.
- q: LLM의 학습 과정인 '사전 학습'과 '미세 조정'은 어떻게 다른가요?
  a: 사전 학습은 대규모 비정형 데이터를 통해 언어의 일반적인 구조와 지식을 익히는 초기 단계입니다. 미세 조정은 특정 작업에 맞춰 모델을 다듬거나 인간의 피드백을 반영하는 강화학습(RLHF)을 통해 답변의 적절성과 정확도를 높이는 최적화 단계입니다.
- q: 토큰화(Tokenization)란 무엇이며 왜 중요한가요?
  a: 토큰화는 텍스트를 모델이 처리할 수 있는 최소 단위인 '토큰'으로 나누는 전처리 과정입니다. 인공지능은 텍스트를 직접 이해하는 것이 아니라 이 토큰화된 숫자를 통해 연산하기 때문에, 효율적인 토큰화는 모델의 성능과 비용에 직접적인 영향을 미칩니다.
- q: LLM과 소형 언어 모델(SLM)의 주요 차이점은 무엇인가요?
  a: LLM은 막대한 자원을 소모하는 범용 모델인 반면, SLM은 파라미터 수를 최적화하여 특정 목적(의료, 법률 등)에 집중한 모델입니다. SLM은 가볍고 빠르기 때문에 보안이 중요한 폐쇄망 환경이나 기기 자체에서 구동되는 온디바이스(On-device) AI를 구현하는 데 더 유리합니다.
- q: LLM 도입 시 발생하는 '환각(Hallucination)' 현상은 어떻게 해결하나요?
  a: 환각은 모델이 잘못된 정보를 사실처럼 말하는 현상으로, 이를 해결하기 위해 RAG(검색 증강 생성) 기술을 주로 활용합니다. 질문과 관련된 신뢰할 수 있는 외부 문서를 먼저 참조한 뒤 답변을 생성하게 함으로써 정보의 정확도를 높일 수 있습니다.
- q: 실무에서 자주 언급되는 RAG(검색 증강 생성)란 구체적으로 무엇인가요?
  a: RAG는 모델이 가진 자체 지식에만 의존하지 않고, 사내 지식 베이스나 외부 데이터베이스에서 관련 정보를 실시간으로 검색하여 답변의 근거로 사용하는 기술입니다. 이를 통해 답변의 신뢰도를 높이고 최신 정보를 실시간으로 반영할 수 있습니다.
- q: 프롬프트 엔지니어링(Prompt Engineering)이 중요한 이유는 무엇인가요?
  a: LLM은 질문(입력값)의 형태에 따라 결과물의 품질이 크게 달라지기 때문입니다. 모델이 맥락을 정확히 이해하도록 질문을 정교하게 설계하고 최적화함으로써, 원하는 형태의 고품질 답변을 더 효율적으로 얻어낼 수 있습니다.
- q: 멀티모달리티(Multimodality)는 LLM의 미래에 어떤 영향을 미치나요?
  a: 멀티모달리티는 텍스트를 넘어 이미지, 소리, 영상 등 다양한 형태의 데이터를 동시에 이해하고 생성하는 능력입니다. 이를 통해 LLM은 시각적 정보나 청각적 맥락까지 결합하여 인간과 더욱 유사한 통합적 사고와 상호작용이 가능해집니다.

### 2. 거대 언어 모델(LLM)의 정의
거대 언어 모델(LLM)은 방대한 양의 텍스트 데이터를 학습해 인간의 언어를 이해하고 생성하도록 설계된 인공지능 모델입니다. 수억 개 이상의 파라미터(매개변수)를 바탕으로 구축되며, 주로 트랜스포머(Transformer) 아키텍처를 활용해 문맥을 파악합니다. 단순한 텍스트 생성을 넘어 요약, 번역, 복잡한 논리 추론 등 고도화된 언어 처리 작업 전반을 수행할 수 있습니다.

### 3. 기술적 등장 배경
LLM의 등장은 기존 RNN(순환 신경망)이나 LSTM 기반 모델이 가진 구조적 한계를 극복하는 과정에서 시작되었습니다. 이전 모델들은 문장이 길어질수록 앞부분의 정보를 소실하는 '장기 의존성' 문제와 순차 연산으로 인한 학습 속도 저하라는 고질적인 문제를 안고 있었거든요. 2017년 구글이 발표한 '어텐션(Attention)' 메커니즘 기반의 트랜스포머 아키텍처는 데이터 병렬 처리를 가능하게 하며 학습 효율을 획기적으로 높였습니다. 여기에 컴퓨팅 자원의 성장과 대규모 인터넷 데이터가 결합되면서 지금의 LLM 시대가 본격화되었습니다.

### 4. 핵심 원리와 주요 특징
*   <b>트랜스포머 아키텍처:</b> 자기 어텐션(Self-Attention) 메커니즘을 통해 문장 내 모든 단어 사이의 관계를 동시에 계산합니다. 이를 통해 물리적으로 멀리 떨어진 단어 간의 맥락도 정확하게 파악할 수 있다는 것이 큰 장점입니다.
*   <b>사전 학습 및 미세 조정:</b> 먼저 비정형 데이터를 통해 언어의 일반적인 구조를 익히는 사전 학습(Pre-training)을 거칩니다. 이후 특정 작업에 맞춰 모델을 다듬는 명령어 튜닝이나, 인간의 피드백을 반영해 답변의 적절성을 높이는 강화학습(RLHF) 과정을 통해 성능을 고도화합니다.
*   <b>확장 법칙과 창발적 능력:</b> 모델의 파라미터 수, 데이터 규모, 계산 성능이 커질수록 모델의 역량이 비약적으로 향상되는 특성이 있습니다. 특히 특정 규모 임계치를 넘어서면 이전에는 없던 고차원적인 추론 능력이 나타나기도 하는데, 업계에서는 이를 '창발적 능력(Emergent abilities)'이라 부릅니다.

### 5. LLM과 SLM의 차이점
LLM이 모든 분야에 능통한 범용 해결사라면, <b>SLM(Small Language Model, 소형 언어 모델)</b>은 특정 목적에 집중한 전문가에 가깝습니다. LLM은 압도적인 성능을 보여주지만 구축과 운영에 막대한 비용과 연산 자원이 소모됩니다. 반면 SLM은 파라미터 수를 최적화하여 특정 도메인(의료, 법률, 금융 등)에서 효율적인 성능을 내도록 설계되죠. 덕분에 가볍고 빠른 실행이 가능하며, 보안이 중요한 폐쇄망 환경이나 온디바이스(On-device) AI 구현에 더 적합한 선택지가 됩니다.

### 6. 실무 적용 및 주요 용어
*   <b>현장 적용 사례:</b> 기업 내부의 지식 베이스를 활용한 챗봇 구축이 대표적입니다. 사용자가 질문하면 <b>RAG(검색 증강 생성)</b> 기술을 통해 사내 문서를 먼저 참조한 뒤 답변을 내놓는 방식이죠. 이는 LLM이 잘못된 정보를 사실처럼 말하는 '환각(Hallucination)' 현상을 줄이고 답변의 신뢰도를 높여줍니다.
*   <b>관련 용어 정리:</b>
    1.  <b>Tokenization:</b> 텍스트를 모델이 처리할 수 있는 최소 단위인 '토큰'으로 나누는 전처리 과정입니다.
    2.  <b>Prompt Engineering:</b> 모델로부터 원하는 결과를 얻기 위해 입력값(질문)을 정교하게 설계하고 최적화하는 기법입니다.
    3.  <b>Multimodality:</b> 텍스트뿐만 아니라 이미지, 소리, 영상 등 서로 다른 형태의 데이터를 동시에 이해하고 생성하는 능력입니다.
