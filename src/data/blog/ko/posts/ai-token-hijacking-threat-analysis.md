최근 사이버 공격은 ID와 비밀번호라는 1차 관문을 넘어, 로그인 세션 그 자체를 노리는 방향으로 진화하고 있습니다. 과거의 해킹이 계정 정보를 알아내는 수준에 머물렀다면, 이제는 이미 인증이 완료된 '토큰'을 직접 탈취해 다요소 인증(MFA) 체계까지 무력화하는 단계에 이르렀죠. 그 중심에는 생성형 AI를 활용해 공격의 정교함과 속도를 높인 'AI 토큰 탈취(AI Token Hijacking)'가 있습니다.

## AI 토큰 탈취, 기업 보안의 새로운 위협

마이크로소프트 보안 연구소(Microsoft Defender Security Research)의 최신 보고서에 따르면, 'EvilTokens'라 불리는 서비스형 피싱(PhaaS) 툴킷을 활용한 대규모 캠페인이 포착되었습니다. 이 공격의 핵심은 자동화된 인프라를 통해 AI 토큰 탈취를 시도한다는 점입니다. 공격자들은 생성형 AI로 피해자의 직무나 워크플로우에 최적화된 맞춤형 미끼를 던져 클릭을 유도합니다.

이렇게 탈취된 토큰은 기업 클라우드 인프라 전체로 위협이 확산되는 발판이 됩니다. 토큰이 유효한 기간 동안 공격자는 별도의 인증 없이도 이메일 무단 반출, 사내 디렉터리 분석, 권한 상승 등을 시도할 수 있습니다. 이는 단순한 데이터 유출을 넘어 기업 내의 근본적인 '신뢰 체인'이 붕괴되는 심각한 상황을 초래합니다.

!![AI 토큰 탈취(AI Token Hijacking) - 대규모 피싱 캠페인을 나타내는, 서버 노드에서 빨간색 경고 아이콘이 깜빡이는 글로벌 네트워크의 개념도, 플랫 디자인 에디토리얼 스타일, 전문적인 컬러 팔레트.](../../../../../source/posts/AI_토큰_탈취(AI_Token_Hijacking)/4cc4071d-1.webp)

## 정상적인 인증 절차를 파고드는 디바이스 코드 피싱

이번에 발견된 공격 수법 중 특히 주의 깊게 살펴봐야 할 대목은 '디바이스 코드 인증 흐름(Device Code Flow)'의 악용입니다. 원래 스마트 TV나 프린터처럼 입력 장치가 제한된 기기에서 편의를 위해 설계된 기능이지만, 공격자는 이 정상적인 프로세스 사이에 침투하여 사용자가 피싱 사이트에서 제공한 코드를 입력하도록 유도합니다.

여기에는 '동적 코드 생성'과 '백엔드 자동화'라는 정교한 기술이 동원됩니다. 일반적으로 15분의 유효 시간을 가지는 디바이스 코드를 우회하기 위해, 공격자들은 Railway.com 같은 PaaS 플랫폼에서 수천 개의 단기 폴링 노드를 생성합니다. 사용자가 링크를 클릭하는 찰나에 실시간으로 코드를 생성해 만료 시간을 피해 가는 방식이죠. 사용자가 코드를 입력하는 즉시 세션이 탈취되며, 공격자의 세션은 사용자의 환경과 분리되어 있어 기존 보안 모니터링으로는 감지하기가 무척 까다롭습니다.

!![AI 토큰 탈취(AI Token Hijacking) - 정상적인 함수에 악성 스크립트가 삽입되고 있는 코드 편집기 화면의 클로즈업, 현대적인 사무실의 부드러운 보케 배경, 디지털 아트 스타일.](../../../../../source/posts/AI_토큰_탈취(AI_Token_Hijacking)/822cf2dd-2.webp)

## 데이터와 명령의 경계를 허무는 프롬프트 인젝션

AI 에이전트의 확산은 또 다른 보안 전선을 형성하고 있습니다. 최근 주요 AI 모델의 에이전트들이 '간접 프롬프트 인젝션(Indirect Prompt Injection)' 공격에 취약하다는 사실이 밝혀졌습니다. 공격자가 AI에게 직접 명령을 내리는 대신, AI가 읽어 들일 외부 데이터(GitHub PR 제목, 이메일 본문 등)에 악의적인 지시문을 숨겨두는 수법입니다.

예를 들어 보안 검토를 수행하는 AI 에이전트가 악성 코드가 포함된 PR 제목을 읽는 순간, 그 안에 숨겨진 명령에 따라 현재 세션의 API 키나 토큰을 공격자 서버로 전송할 수 있습니다. AI 모델이 '데이터'와 '명령'을 명확하게 구분하지 못하는 구조적 한계 때문이죠. 사용자는 AI가 업무를 수행하고 있다고 믿지만, 이면에서는 조용히 토큰 탈취가 진행되는 셈입니다. 이러한 취약점이 공식적인 CVE 발급 없이 암묵적으로 패치되는 경우가 많아, 구버전 에이전트를 사용하는 환경은 위험에 그대로 노출될 가능성이 큽니다.

!![AI 토큰 탈취(AI Token Hijacking) - 회로 패턴이 통합된 보안 금고 문, 데이터 보호와 제로 트러스트 아키텍처를 상징함, 3D 렌더링, 깔끔한 건축 스타일.](../../../../../source/posts/AI_토큰_탈취(AI_Token_Hijacking)/24e52313-3.webp)

## 네트워크 레이어에서의 실시간 대응과 방어 전략

AI 토큰 탈취에 대응하기 위해서는 개별 사용자의 주의력에만 의존해서는 안 됩니다. 네트워크 레이어에서부터 철저한 검증과 감시가 이루어지는 다층 방어 전략이 필수적입니다.

이러한 관점에서 **하이온넷(Haionnet)**의 보안 솔루션은 실무적인 방어막을 제공합니다. 전용 회선 기반의 안정적인 인프라를 바탕으로, 통합 보안 관제(MSSP)를 통해 비정상적인 트래픽 흐름을 실시간으로 추적합니다. 공격자가 주로 사용하는 인프라로의 데이터 유출이나 의심스러운 도메인으로부터의 접근을 네트워크 단에서 원천 차단함으로써 1차 방어선을 형성하거든요. 특히 기업의 인프라 특성에 맞춘 정교한 정책 설정이 가능해, 자동화된 AI 공격으로부터 비즈니스의 연속성을 확보하는 데 중추적인 역할을 합니다.

> "AI는 보안의 강력한 우군이지만, 동시에 공격자에게는 그 어느 때보다 날카로운 창이 되었습니다. 이제는 로그인 이후의 '세션 신뢰성'을 어떻게 담보할 것인가가 보안의 성패를 가를 것입니다."

## 지속적인 검증, 제로 트러스트로의 전환

AI 토큰 탈취는 이제 이론적인 시나리오가 아닌 현실의 위협입니다. 자동화된 도구들이 끊임없이 세션을 노리고 있으며, 우리가 업무 효율을 위해 도입한 AI 에이전트가 오히려 공격의 통로가 되기도 합니다.

이제 '한 번 인증된 세션은 안전하다'는 믿음을 거둘 때입니다. 모든 연결을 끊임없이 검증하고, 네트워크 수준에서 강력한 보안 통제를 병행하는 제로 트러스트 원칙을 실천해야 합니다. 하이온넷과 같은 보안 파트너와 함께 인프라의 기초를 다시 점검하는 것, 그것이 AI 시대를 안전하게 헤쳐 나가는 가장 확실한 방법입니다.

  <div class="faq-content">

스마트 TV 등에서 쓰이는 정상적인 인증 절차를 악용하기 때문입니다. 공격자는 실시간 자동화 노드를 생성해 코드 만료 시간을 우회하며, 사용자가 코드를 입력하는 즉시 세션을 가로챕니다. 이는 기존 보안 모니터링으로 감지가 매우 어렵습니다.

  </div>
</details>

<details>
  <summary>간접 프롬프트 인젝션(Indirect Prompt Injection)이란 무엇인가요?</summary>
  <div class="faq-content">

AI가 읽어 들이는 외부 데이터(이메일, 문서 등)에 악의적인 지시문을 숨겨두는 수법입니다. AI 모델이 데이터와 명령을 명확히 구분하지 못하는 구조적 한계를 노려, AI가 스스로 세션 토큰이나 API 키를 공격자에게 전송하게 만듭니다.

  </div>
</details>

<details>
  <summary>왜 로그인 이후의 &#x27;세션 보안&#x27;이 중요해졌나요?</summary>

  </div>
</details>

AI 에이전트의 취약점은 공식적인 CVE 발급 없이 암묵적으로 패치되는 경우가 많습니다. 따라서 구버전 에이전트를 사용하지 않도록 주의해야 하며, AI가 처리하는 외부 데이터가 명령어로 돌변할 수 있다는 점을 항상 경계해야 합니다.

  </div>
</details>

  </div>
</details>
