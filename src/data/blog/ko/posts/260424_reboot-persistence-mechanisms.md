    실행되게 만듭니다.

    회피합니다.

    구분이 어렵습니다.

  a: 운영체제의 API를 호출하지 않고 Offreg.dll 같은 라이브러리로 레지스트리 하이브 파일을 직접 수정합니다. API 호출 기록이 남지
    않아 실시간 감시를 피할 수 있습니다.

    의심 징후가 나타나지 않습니다.

  a: MFT 타임스탬프와 USN 저널의 변경 기록을 대조하여 파일 생성 시점을 확인하고, Sysmon의 레지스트리 이벤트와 윈도우 보안 로그를

  a: 운영체제보다 먼저 로드되는 커널 모드 드라이버나 UEFI 부트킷을 이용하는 등, 소프트웨어 보안 솔루션의 탐지 범위를 벗어나는 하드웨어 레벨로

  a: 단순 차단을 넘어 파편화된 침해 지표들을 연결해 전체 시나리오를 복구해야 합니다. 시스템 구조를 깊이 이해하고 하드웨어 기반의 신뢰 루트를
    확보하는 것이 중요합니다.

침투에 성공한 공격자에게 주어진 가장 시급한 과제는 역설적으로 '생존'입니다. 초기 진입에 성공하더라도 시스템이 재부팅되거나 사용자가 로그아웃하면 확보한 권한이 소멸하기 때문입니다. 따라서 공격자는 시스템 환경 변화에 상관없이 자신의 권한을 유지하기 위한 기술적 장치를 마련하는데, 이를 '영속성(Persistence) 메커니즘'이라 합니다. 보안 사고 분석에서 이 영속성 흔적을 추적하고 대응하는 역량은 위협의 깊이를 파악하는 핵심 지표가 됩니다.

최근의 공격 양상은 단순히 실행 파일을 숨기는 수준을 넘어, 윈도우 운영체제의 정상적인 관리 기능을 정교하게 악용하는 방향으로 고도화되고 있습니다. 시스템 내부의 합법적인 경로를 침해의 거점으로 삼는 방식입니다.

### 합법적 기능의 이면에 숨은 자동 실행의 덫

전통적인 방식의 공격은 윈도우 레지스트리의 자동 실행 키를 수정하는 것에서 시작합니다. 보안 관리자에게 친숙한 `Run`이나 `RunOnce` 경로에 악성 페이로드를 등록하는 방식입니다. 하지만 최근에는 파일 시스템에 흔적을 남기지 않는 파일리스(Fileless) 기법이 결합하며 탐지를 더욱 어렵게 만들고 있습니다. 레지스트리 값 자체에 파워셸(PowerShell)이나 스크립트 코드를 직접 삽입하여, 시스템이 시작될 때 파일 실행 없이 메모리상에서 즉각 동작하도록 설계하는 식입니다.

예약된 작업(Scheduled Tasks) 기능 역시 단골 소재입니다. 공격자는 특정 시간대나 시스템 이벤트에 맞춰 동작하도록 작업을 등록함으로써 부팅 직후 발생하는 보안 솔루션의 집중 감시 시간대를 회피합니다. 분석가는 `C:\Windows\System32\Tasks` 내의 XML 명세와 레지스트리의 `TaskCache` 정보를 대조하여 인가되지 않은 불일치 사례를 찾아내는 정밀한 검증 과정을 거쳐야 합니다.

### EDR의 감시망을 우회하는 로우 레벨 접근

엔드포인트 탐지 및 대응(EDR) 솔루션의 발달에 대응하여, 공격자들은 운영체제 API를 거치지 않는 우회로를 찾고 있습니다. 대표적인 사례가 `Offreg.dll`과 같은 라이브러리를 활용해 레지스트리 하이브 파일을 직접 수정하는 방식입니다. 실시간 API 호출을 감시하는 보안 솔루션 입장에서는 호출 기록이 남지 않으므로 상태 변화를 감지하기 매우 어렵습니다.

특히 필수 사용자 프로필인 `NTUSER.MAN`을 악용하는 기법은 보안 분석에서 주목해야 할 대목입니다. 시스템은 로그인 과정에서 `NTUSER.DAT`보다 `NTUSER.MAN` 파일을 우선적으로 로드하는 특성이 있습니다. 공격자가 오프라인 상태에서 이 파일을 수정해 배치하면, 재부팅 전까지는 아무런 징후가 나타나지 않다가 사용자가 다시 로그인하는 순간 악성 설정이 활성화됩니다. 이는 단순한 로그 분석만으로는 포렌식 영속성을 완벽히 파악할 수 없음을 시사합니다.

### 파편화된 침해 지표를 연결하는 분석적 관점

공격자가 영속성을 확보하기 위해 수행하는 모든 행위는 결국 미세한 흔적을 남깁니다. 유능한 분석가는 단편적인 정보에 의존하지 않고 여러 지표를 교차 검증하여 전체 맥락을 재구성합니다.

- 마스터 파일 테이블(MFT)의 타임스탬프와 USN 저널(UsnJrnl)의 변경 기록을 대조하여 파일 생성과 레지스트리 수정의 인과관계 파악
- 윈도우 이벤트 로그 중 객체 액세스(Security 4663)와 레지스트리 값 설정(Sysmon 13) 이벤트를 연계 분석
- THOR와 같은 스캐너로 시스템 전반의 이상 징후를 식별하고, Timesketch 등의 도구를 활용해 타임라인 기반의 공격 흐름 복구

영속성 분석은 흩어진 점들을 선으로 연결하여 공격자의 시나리오를 복구하는 과정입니다. 시스템 구석구석에 숨겨진 비정상적인 연결 고리를 찾아내는 것이 분석의 핵심입니다.

### 하드웨어와 커널 수준으로 깊어지는 위협의 미래

앞으로의 위협은 운영체제보다 먼저 로드되는 커널 모드 드라이버나 UEFI 부트킷을 이용한 로우 레벨 영속성으로 진화할 가능성이 큽니다. 소프트웨어 기반 보안 솔루션의 탐지 범위를 벗어나는 영역인 만큼, 하드웨어 기반의 신뢰 루트(Root of Trust) 확보가 필수적인 과제로 떠오르고 있습니다.

결국 보안의 성패는 침입 시도를 막는 것을 넘어, 그들이 시스템에 뿌리를 내리는 영속성 단계에서 얼마나 신속하게 연결 고리를 끊어내느냐에 달려 있습니다. 도구와 기법은 계속해서 변하겠지만, 시스템 구조를 깊이 이해하고 미세한 변화를 감지하는 분석적 통찰력은 디지털 자산을 지키는 가장 강력한 방어 기제가 될 것입니다.
