Q1: 영속성 메커니즘이란 무엇인가요?
A1: 공격자가 초기 침투에 성공한 뒤, 시스템 재부팅이나 로그아웃과 같은 환경 변화에 상관없이 확보한 권한을 계속 유지하기 위해 사용하는 기술적 장치를 말합니다.

Q2: 공격자들이 영속성을 확보하려는 가장 큰 이유는 무엇인가요?
A2: 시스템이 재부팅되면 메모리상에서 동작하던 공격 권한이 소멸하기 때문입니다. 공격자는 지속적인 정보 탈취와 추가 침투를 위해 생존 수단을 마련해야 합니다.

Q3: 윈도우 시스템에서 가장 흔히 악용되는 영속성 경로는 어디인가요?
A3: 주로 레지스트리의 자동 실행 키인 Run이나 RunOnce 경로를 수정하거나, 시스템 관리 기능인 예약된 작업을 생성하여 악성 코드가 자동으로 실행되게 만듭니다.

Q4: 파일리스 기법이 결합된 영속성 공격의 특징은 무엇인가요?
A4: 실제 실행 파일을 생성하는 대신 레지스트리 값 자체에 파워셸 스크립트를 삽입합니다. 파일 시스템에 흔적을 남기지 않고 메모리에서 동작해 탐지를 회피합니다.

Q5: 예약된 작업을 이용한 공격이 탐지하기 어려운 이유는 무엇인가요?
A5: 특정 시간대나 이벤트를 설정해 부팅 직후의 집중 감시 시간을 피할 수 있기 때문입니다. 정상적인 시스템 관리 작업으로 위장하는 경우가 많아 구분이 어렵습니다.

Q6: EDR 솔루션을 우회하기 위해 공격자가 사용하는 로우 레벨 접근 방식은 무엇인가요?
A6: 운영체제의 API를 호출하지 않고 Offreg.dll 같은 라이브러리로 레지스트리 하이브 파일을 직접 수정합니다. API 호출 기록이 남지 않아 실시간 감시를 피할 수 있습니다.

Q7: NTUSER.MAN 파일을 악용한 영속성 기법의 원리는 무엇인가요?
A7: 윈도우 로그인 시 NTUSER.DAT보다 우선 로드되는 사용자 프로필의 특성을 이용합니다. 오프라인 상태에서 이 파일을 수정하면 로그인 전까지는 의심 징후가 나타나지 않습니다.

Q8: 보안 분석 시 영속성 흔적을 찾기 위해 대조해야 할 핵심 지표는 무엇인가요?
A8: MFT 타임스탬프와 USN 저널의 변경 기록을 대조하여 파일 생성 시점을 확인하고, Sysmon의 레지스트리 이벤트와 윈도우 보안 로그를 연계하여 분석해야 합니다.

Q9: 향후 영속성 위협은 어떤 형태로 더욱 고도화될 것으로 예상되나요?
A9: 운영체제보다 먼저 로드되는 커널 모드 드라이버나 UEFI 부트킷을 이용하는 등, 소프트웨어 보안 솔루션의 탐지 범위를 벗어나는 하드웨어 레벨로 진화할 가능성이 큽니다.

Q10: 고도화된 영속성 공격에 대응하기 위한 효과적인 방어 전략은 무엇인가요?
A10: 단순 차단을 넘어 파편화된 침해 지표들을 연결해 전체 시나리오를 복구해야 합니다. 시스템 구조를 깊이 이해하고 하드웨어 기반의 신뢰 루트를 확보하는 것이 중요합니다.
