Q1: Model Context Protocol(MCP)이란 무엇인가요?
A1: MCP는 앤스로픽이 공개한 오픈 표준으로, AI 모델과 외부 데이터 시스템 간의 연결을 표준화하는 프로토콜입니다. 파편화된 인터페이스를 단일 규격으로 통합하여 AI가 실시간 데이터와 외부 도구에 쉽게 접근할 수 있게 돕습니다.

Q2: MCP가 왜 USB-C 규격에 비유되나요?
A2: 과거 기기마다 제각각이었던 충전 규격을 USB-C가 하나로 통합한 것처럼, MCP는 AI 에이전트가 데이터베이스나 API에 접근하는 경로를 하나의 표준으로 묶어주기 때문입니다. 이를 통해 복잡한 연결 과정을 획기적으로 단순화합니다.

Q3: MCP를 구성하는 세 가지 핵심 요소는 무엇인가요?
A3: 읽기 전용 데이터 소스인 리소스(Resources), 이메일 발송 등 직접 기능을 수행하는 도구(Tools), 그리고 모델의 도구 활용을 안내하는 지침인 프롬프트(Prompts)로 구성됩니다. 이 세 요소가 유기적으로 작동하여 AI의 행동을 지원합니다.

Q4: MCP 도입이 개발 생산성에 어떤 도움을 주나요?
A4: 기존에는 N개의 앱을 M개의 서비스에 연결할 때 N*M개의 작업이 필요했지만, MCP는 각 시스템이 표준을 한 번만 구현하면 되는 N+M 구조로 전환합니다. 결과적으로 전체 통합 복잡도와 유지보수 비용이 기하급수적으로 감소합니다.

Q5: MCP는 AI의 어떤 한계를 극복하려 하나요?
A5: AI 모델이 학습 데이터에만 의존하는 지식 동결 현상과 외부 시스템과 소통하지 못하는 폐쇄성을 해결합니다. 실시간 정보를 확인하고 외부 작업을 직접 수행하는 범용 인터페이스 역할을 통해 고립된 지능을 세상과 연결합니다.

Q6: 기존 RAG 기술과 MCP의 결정적인 차이점은 무엇인가요?
A6: RAG는 벡터 DB에 저장된 과거 지식을 검색하는 데 집중하는 반면, MCP는 라이브 API와 DB에 직접 연결되어 현재 시점의 데이터를 가져오고 행동합니다. RAG가 참고 서적을 찾는 것이라면 MCP는 실시간 통화와 직접 조치에 가깝습니다.

Q7: MCP 서버의 자율적 탐색 기능이란 무엇인가요?
A7: 개발자가 특정 API를 하드코딩하지 않아도, AI가 런타임에 MCP 서버가 노출한 기능 명세서를 실시간으로 확인하고 필요한 도구를 스스로 판단하여 호출하는 방식입니다. 이는 정해진 연결을 넘어 지능형 통합을 가능하게 하는 핵심입니다.

Q8: MCP 환경에서 발생할 수 있는 주요 보안 위협은 무엇인가요?
A8: AI가 외부 시스템에서 코드를 실행하거나 데이터를 조작할 수 있으므로, 프롬프트 주입을 통해 권한 밖의 명령을 수행하게 만드는 시나리오가 위험합니다. 적절한 통제 장치가 없다면 시스템 무력화나 데이터 유출 위협에 노출될 수 있습니다.

Q9: 실무 현장에서의 보안 관리 및 설계 방안은 무엇인가요?
A9: 로컬에서는 표준 입출력을 통해 통신 경로를 제한하고, 외부 연동 시에는 OAuth2 기반 권한 제어로 최소 권한 원칙을 준수해야 합니다. 또한 런타임 위험 요소를 사전에 감지하는 스캐닝 프로세스를 도입하는 설계가 필수적입니다.

Q10: MCP의 기술적 통신 방식과 구조적 특징은 무엇인가요?
A10: JSON-RPC 2.0 기반의 클라이언트-서버 아키텍처를 채택하고 있습니다. 단순한 일회성 요청을 넘어 상태 유지가 가능한 양방향 통신을 지원하여, AI 에이전트가 복잡한 작업 흐름 속에서 지속적으로 맥락을 유지하며 상호작용할 수 있습니다.
