Q1: MCP(Model Context Protocol)란 무엇인가요?
A1: 앤스로픽이 제안한 개방형 표준으로, AI 모델과 외부 데이터 및 도구를 연결하는 통로입니다. AI가 실시간 정보에 접근하고 직접 행동할 수 있도록 돕는 '통합 인터페이스' 역할을 합니다.

Q2: MCP를 'AI를 위한 USB-C'라고 부르는 이유는 무엇인가요?
A2: 모델과 도구가 각각 전용 통합 코드를 짤 필요 없이, MCP라는 단일 규격으로 즉시 연결되기 때문입니다. 복잡한 N×M 연동 구조를 N+M 수준으로 단순화하여 범용적인 호환성을 제공합니다.

Q3: MCP를 구성하는 3가지 핵심 요소는 무엇인가요?
A3: AI가 읽기 전용으로 참조하는 '리소스(Resources)', 모델이 권한을 위임받아 실행하는 액션인 '도구(Tools)', 그리고 모델의 사고 가이드를 제공하는 '프롬프트(Prompts)'입니다.

Q4: MCP의 기술적 기반인 JSON-RPC 2.0은 어떤 역할을 하나요?
A4: 클라이언트와 서버 간의 양방향 스트리밍 세션을 유지합니다. 일회성인 REST API와 달리 상태를 유지하므로, AI 에이전트가 실시간 상황을 추적하고 복잡한 작업을 유연하게 수행하게 합니다.

Q5: MCP는 AI의 어떤 한계를 해결하나요?
A5: 학습 데이터 시점에 멈춰있는 '지식의 고립'과 외부와 소통할 수 없는 '기능적 폐쇄성'을 해결합니다. 이를 통해 AI는 정적인 챗봇을 넘어 실시간 데이터를 다루는 지능형 에이전트로 진화합니다.

Q6: 기존 RAG(검색 증강 생성) 방식이 있는데 왜 MCP가 필요한가요?
A6: RAG는 정적인 대규모 문서 검색에 유리하지만, MCP는 실시간 시장 지표나 최신 코드 변경 이력 같은 '동적 데이터'를 다루는 데 특화되어 있습니다. 두 방식은 상호 보완적인 하이브리드 전략으로 쓰입니다.

Q7: 개발자 입장에서 MCP 도입 시 가장 큰 장점은 무엇인가요?
A7: 중복되는 통합 작업에서 해방됩니다. MCP 서버를 한 번만 구축하면, 이 표준을 지원하는 모든 AI 에이전트에서 추가 작업 없이 즉시 데이터와 기능을 활용할 수 있어 개발 효율이 극대화됩니다.

Q8: MCP 환경에서 보안과 데이터 통제는 어떻게 이루어지나요?
A8: '사용자의 명시적 승인' 없이는 도구 실행이 불가능합니다. 모든 활동은 감사 로그로 기록되며, 엄격한 접근 제어 목록(ACL)을 통해 민감한 정보에 대한 접근 권한을 중앙에서 효율적으로 통제합니다.

Q9: MCP가 기업형 AI 인프라의 표준이 될 것으로 보는 이유는 무엇인가요?
A9: 구글, MS, 오픈AI 등 주요 테크 기업들이 생태계에 합류하고 있기 때문입니다. 데이터 사일로를 허물면서도 보안을 유지할 수 있어 금융, 의료 등 전문 분야의 폐쇄형 AI 구축에 필수적입니다.

Q10: 에이전틱 AI 시대를 준비하는 조직에 MCP는 어떤 의미인가요?
A10: AI 에이전트가 현실 세계와 상호작용하기 위한 '감각 기관'이자 '손발'이 됩니다. MCP 표준을 선제적으로 수용하는 조직은 복잡한 API 연동 노동에서 벗어나 AI 비즈니스 로직 개발에만 집중할 수 있습니다.
