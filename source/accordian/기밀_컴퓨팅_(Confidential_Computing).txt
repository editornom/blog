Q1: 기밀 컴퓨팅이란 무엇인가요?
A1: 하드웨어 기반의 신뢰 실행 환경(TEE)을 통해 데이터가 처리되는 동안에도 이를 보호하는 기술입니다. CPU 내에 격리된 공간을 만들어 메모리상의 데이터를 암호화하고 무결성을 유지합니다.

Q2: 기밀 컴퓨팅이 왜 중요한가요?
A2: 기존의 저장 및 전송 중 암호화와 달리, 데이터 연산을 위해 복호화된 상태에서도 보안을 유지하기 때문입니다. 이를 통해 OS나 관리자 권한을 가진 공격자로부터 데이터를 지킬 수 있습니다.

Q3: TEE의 핵심적인 역할은 무엇인가요?
A3: CPU 내부에 일반 프로세스와 완전히 분리된 독립 보안 영역인 엔클레이브를 생성합니다. 인가된 코드만이 하드웨어 레벨에서 데이터를 복호화해 실행할 수 있도록 물리적으로 격리하는 역할을 합니다.

Q4: '처리 중인 데이터' 보안은 어떤 의미인가요?
A4: 메모리에 로드되어 실제 연산이 일어나는 시점의 데이터를 보호한다는 의미입니다. 그동안 보안의 공백이었던 런타임 환경에서도 데이터의 기밀성과 무결성을 확보할 수 있게 됩니다.

Q5: 원격 증명이란 무엇인가요?
A5: 데이터 소유자가 서버로 워크로드를 보내기 전, 해당 하드웨어의 신뢰성과 소프트웨어 변조 여부를 암호학적으로 검증하는 절차입니다. 객관적인 수치로 시스템의 무결성을 확인할 수 있습니다.

Q6: 기존의 암호화 방식과는 어떤 차이가 있나요?
A6: 기존 방식은 데이터가 저장되거나 이동할 때만 보호하지만, 기밀 컴퓨팅은 데이터가 사용되는 순간까지 보호합니다. 특히 인프라 관리자조차 데이터에 접근할 수 없다는 점이 가장 큰 차별점입니다.

Q7: 퍼블릭 클라우드에서도 안전을 보장할 수 있나요?
A7: 그렇습니다. 하드웨어 수준에서 강제적인 격리가 이루어지므로 클라우드 서비스 제공자나 호스트 운영체제조차 엔클레이브 내부 데이터에 접근할 수 없습니다. 기업은 통제권 밖에서도 보안을 유지합니다.

Q8: 데이터 클린 룸 서비스에 어떻게 활용되나요?
A8: 여러 기관이 원본 데이터를 노출하지 않고 공동 분석을 진행할 때 활용됩니다. 기밀 컴퓨팅 환경 내에서 데이터를 결합해 학습하되, 각 참여자는 상대방의 데이터에 접근하지 못하고 오직 결과물만 공유받습니다.

Q9: AI 모델 보호에는 어떤 이점이 있나요?
A9: 수조 원의 가치를 지닌 AI 모델의 아키텍처와 가중치가 추론 과정에서 유출되는 것을 방지합니다. 모델 전체를 기밀 컴퓨팅 환경에 적재하여 실행함으로써 독점적인 지적 재산을 안전하게 보호할 수 있습니다.

Q10: 기밀 컴퓨팅이 지향하는 보안의 최종 형태는 무엇인가요?
A10: 제로 트러스트 원칙을 컴퓨팅 인프라의 심부인 하드웨어 레벨까지 확장하는 것입니다. 데이터 활용의 전 과정에서 완벽한 통제권을 제공하여 보안이 비즈니스의 가속기로 기능하게 만드는 것을 지향합니다.
